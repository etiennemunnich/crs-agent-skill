# WAF Rule Management — Example Outputs

Example rule files and go-ftw test suites generated by the skill scripts. Use these as reference when authoring your own rules and tests.

## Rule Examples

| File | Description |
|------|-------------|
| **REQUEST-100001-GRAPHQL-EXCLUSION-RUNTIME.conf** | Runtime exclusion for GraphQL false positives (rule 942100). Generated with `generate_exclusion.py` — scoped to `/api/graphql`, uses `ctl:ruleRemoveById` placed before CRS include. |
| **rules-api-images-pathtrav.conf** | Custom path traversal rule for `/api/images/` endpoint. Covers unencoded and URL-encoded variants (`../`, `%2e%2e%2f`). Includes go-ftw test reference. |
| **rules-payment-api.conf** | Positive-security rules from OpenAPI spec. Generated with `openapi_to_rules.py` — path allowlist, Content-Type and Authorization enforcement for payment API. |
| **rules-payment-api-allow.conf** | Allowlist variant of payment API rules — alternative structure for positive security model. |

| File | Description |

## Generating Your Own

```bash
# OpenAPI → WAF rules
python scripts/openapi_to_rules.py openapi.yaml -o rules.conf

# Exclusion for false positive
python scripts/generate_exclusion.py --rule-id 942100 --uri /api/graphql --type runtime -o exclusion.conf

# go-ftw test scaffold
python scripts/generate_ftw_test.py rule.conf -o tests.yaml
```
